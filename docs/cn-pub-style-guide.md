# 公众号样式指南(content-factory v2.5 升级)

> **为什么需要**:Markdown 在公众号编辑器里部分语法不支持(表格 / 代码块 / 部分 emoji),
> 直接复制粘贴会让文章视觉割裂、信息密度高但**呼吸感差**。
> 这份指南是 9 篇深度文实战沉淀的"公众号友好"排版规则。
>
> ⚠️ 本指南提及的 `scripts/cn-pub-beautify.py` 已被 monorepo v1.0.0 标记本地保留。
> 等价功能未来迁至 [packages/domains/image](../../packages/domains/image/content_factory_image/__init__.py)。
> 自动化可临时跑 `uv run python -c "from content_factory_image import beautify_article; beautify_article('posts/<slug>.md')"`(功能未上架前请用 markdown 手工美化)。

## 一、核心问题:Mars 到公众号的"翻译损失"

| Markdown 语法 | 公众号支持 | 视觉损失 | 解决方式 |
|---|---|---|---|
| 表格 `\| ... \|` | ❌ 几乎不支持,变无格式文本 | 高 | **转卡片化**:用 `###` + emoji + 加粗 + 列表代替 |
| 代码块 ` ``` ` | ❌ 几乎不可用 | 高 | **转引言**:用 `>` 引用块 + 等宽格式代替 |
| 脚注 `[^1]` | ❌ 不支持 | 中 | **转文内引用**:用 `(来源:xxx)` 代替 |
| 任务列表 `- [ ]` | ❌ 不支持 | 低 | **转 emoji 列表**:`✅` `⚠️` `❌` |
| 部分 emoji (如复杂符号) | ⚠️ 部分显示成方块 | 中 | **用公众号安全 emoji**:`📌⭐🚨🔻🟢🟡🔴💡📊🏷️` 等通用符号 |
| `**加粗**` | ✅ 支持 | 0 | 直接用 |
| `# 标题` | ✅ 支持 | 0 | 直接用 |
| `> 引用` | ✅ 支持 | 0 | 直接用 |
| 链接 `[text](url)` | ⚠️ 链接在公众号编辑器被改写 | 中 | **写正文时只放文末参考**,不放正文链接 |
| 图片 `![](url)` | ❌ 需先上传到素材库 | 高 | **手动上传**:publish.sh 提示里要求 |

## 四、自动化:`scripts/cn-pub-beautify.py`

10 条规则里有 5 条可以自动应用:

| 规则 | 自动化 | 实现 |
|---|---|---|
| 规则 1 TL;DR 摘要卡片 | 半自动(检测摘要缺失时警告) | regex |
| 规则 2 关键数字 emoji | **自动** | regex 匹配 `数字%` / `数字亿` |
| 规则 3 表格 → 卡片 | **自动** | 解析 markdown 表格 → 拆成子标题 + 列表 |
| 规则 4 反共识 emoji | 半自动 | regex 检测"反共识 N"加 🔻 |
| 规则 5 风险 emoji | **自动** | regex 检测"主要风险"段加 🚨 |
| 规则 6 数据快照日期 | **自动** | regex 检测裸日期加"数据快照"标记 |
| 规则 7 段落长度 | 警告(不自动拆) | 检测段落行数,>3 行时警告 |
| 规则 8 章节钩子 | 警告(不自动加) | 检测 `##` 章节是否有钩子段 |
| 规则 9 互动引导 | 警告(不自动加) | 检测文末是否有互动段 |
| 规则 10 中英文 / 数字空格 | **自动** | regex 在中文 + 数字间加空格 |

**使用方式**:

```bash
python3 scripts/cn-pub-beautify.py drafts/posts/glass-bridge-cpo.md
# 输出:drafts/posts/glass-bridge-cpo.styled.md
# diff 一眼能看出改了什么,人工 review 后再覆盖
```
