## 三、选题与 MOC 匹配查询

当 feed 候选选题下来时，用这个查询判断匹配度：

```python
# 选题 MOC 匹配查询（伪代码）
def match_moc(title):
    keywords = {
        'MOC-AI算力': ['AI', '算力', 'GPU', 'HBM', '光模块', '液冷', '数据中心', '铜连接', 'CPO', '燃机'],
        'MOC-半导体国产替代': ['半导体', '国产替代', '光刻', '刻蚀', '硅片', '封装', 'HBM'],
        'MOC-周期资源': ['稀土', '铜', '铝', '黄金', '白银', '煤炭', '化工', '钨', '钼'],
        'MOC-人形机器人': ['机器人', '减速器', '伺服', '传感器', '具身智能', '宇树', '智元', 'Optimus'],
        'MOC-消费零售': ['消费', '零售', '白酒', '美护', '宠物', '潮玩', '零食'],
        'MOC-医药创新药': ['医药', '创新药', 'ADC', 'GLP-1', 'CXO', '出海'],
        'MOC-商业航天': ['航天', '火箭', '卫星', '低轨', '星座'],
        'MOC-汽车智驾': ['智驾', '激光雷达', 'Robotaxi', '域控', '端到端'],
    }
    matches = []
    for moc, kws in keywords.items():
        for kw in kws:
            if kw in title:
                matches.append(moc)
                break
    return matches
```

## 四、选题与 zettaranc-knowledge 联动

zettaranc-knowledge 是用户的 A 股交易体系（"知行小菜鸟"），包含：

- 88 个概念（白线黄线、B1/B2/B3、S1、四不原则等）
- 6 个实体（资金画像）
- 8 个来源摘要
- 2 个操作手册（短线/长线）

**联动策略**：

| 内容方向 | 联动方式 |
|---|---|
| 投资方法论类文章 | 直接引用 zettaranc 框架 |
| 产业链研究类文章 | zettaranc 提供"如何看"框架，Obsidian MOC 提供"看什么"素材 |
| 复盘类文章 | zettaranc 提供 B1/B2/B3 进场逻辑 |
| 风控类文章 | zettaranc 提供"四不原则 + 防守哲学" |

**注意**：之前 v2 标准已经把 zettaranc 压到 ≤ 3 处关键词，主要用于"个人判断"。但**心法层**（如"穿越牛熊靠防守"）仍然可以引用，作为价值主张。

## 五、本路由表使用流程

```
1. 选题时：先查本路由表，看候选选题命中哪个 MOC
2. 研究时：/query 跑相关 MOC，拿 5-10 个 sources
3. 写作时：source 列表里至少有 3-5 个来自 MOC sources
4. 配图时：可以引用 MOC 的概念结构做 mermaid 流程图
5. 归档时：文章 frontmatter 里 tag = MOC 名称
```
