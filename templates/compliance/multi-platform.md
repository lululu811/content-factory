# 多平台合规差异

> 同一个观点在不同平台的合规边界不同。

---

| 平台 | 重点位置 | 风险等级 | 备注 |
|---|---|---|---|
| **公众号** | 全文末尾 + 简介 | 🔴 高 | 读者维权意识最强，截图传播广 |
| **知乎** | 答案开头 + 末尾 | 🔴 高 | 专业认证体系，监管严 |
| **小红书** | 笔记开头 | 🟡 中 | 易截图传播，女性用户维权意识强 |
| **即刻** | 动态末尾 | 🟡 中 | 用户较理性 |
| **B 站** | 视频简介 + 片尾 | 🔴 高 | 视频易 "说错话"，UP 主连带 |
| **Twitter/X** | thread 第 1 条 | 🔴 极高 | SEC 监管，荐股可被起诉 |
| **视频号** | 描述 + 片尾 | 🔴 高 | 微信生态监管严 |

---

## 各平台具体策略

### 公众号

- 全文末尾：中版本或长版本免责声明
- 简介：含 "不构成投资建议"
- 不放个人联系方式（用邮箱或"添加请备注来源"）
- 文章内不放二维码

### 知乎

- 答案开头 1 行免责
- 答案末尾 1 段免责
- 涉及具体标的时，必须标注 "仅为研究案例"
- 避免 "我推荐" "我认为应该买"

### 小红书

- 笔记第 1 句话就要有 "投资有风险" 提示
- 不发"实盘截图"（会被认定为带单）
- 不发"收益对比图"（会被认定为收益承诺）

### 即刻

- 动态末尾 1 行免责
- 不发"目标价"等绝对化判断
- 即刻用户理性，但扩散快

### B 站

- 视频简介必含免责声明
- 片尾字幕必含免责声明
- UP 主连带 → 视频表达更严
- 不放"代码 / 标的"在视频画面正中

### Twitter/X

- 第 1 条 thread 必含免责
- 不用 $TICKER + 数字（避免荐股嫌疑）
- 如要做英文圈，注册 "Research / Information" 类账号，不做 "Investment Advisor" 类

---

## 跨平台同步 SOP

每篇文章发布时：

1. **先发公众号**（你最严格）
2. **2-3 天后同步知乎**（长尾流量）
3. **当天/隔天发即刻 / 小红书**（短内容钩子）
4. **B 站 / 视频号**（如做视频则单独立项）

每个平台版本都要过当地合规清单，不要直接复制粘贴。
